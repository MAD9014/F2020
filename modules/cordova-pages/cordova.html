# Introduction to Cordova

# Cordova | PhoneGap History

In 2011 Nitobi, the company that created PhoneGap, released the PhoneGap project to the Apache Open Source Foundation. The open source project is known as Cordova. The commerically available version is still known as PhoneGap. However, you will often see both names used interchangeably. PhoneGap was the original name and that is still what most people use to refer to the project.

The week after PhoneGap was released to the Apache Foundation Adobe announced its purchase of Nitobi and that the employees of Nitobi would be focused on continuing the development of PhoneGap / Cordova.

PhoneGap website

Cordova website

After Adobe acquired Nitobi and all their technical expertise on PhoneGap they immediately set to work integrating the PhoneGap technology into Dreamweaver. Now we can actually build our web app in Dreamweaver and then package it as "native" PhoneGap application directly from Dreamweaver. For professional development this commercial (paid) option is useful.

For learning and open-source projects, the free and open-source Cordova branch is ideal. We will be using Cordova for our work.

# What is a Cordova / PhoneGap Application?

Here is a great video from 2012 with Brian Leroux (one of the PhoneGap creators) speaking to an HTML5 users group meeting. It explains what HTML5 is, what the Web APIs are, and what PhoneGap is.

And here is a video tutorial on what Cordova is and how you can quickly create a Cordova project and launch it.

With Cordova / PhoneGap you can take a mobile web app (a website built with HTML, CSS, and Javascript) compressed into only a few or even one HTML file, making sure that it is optimized for use on a mobile device and then, through the magic of the PhoneGap program, convert the HTML, CSS and Javascript into a native iOS, Android, Bada, BlackBerry, Symbian, WebOS, or Windows Phone application. Later on we will get back to how PhoneGap actually achieves this. For now, PhoneGap is just our wonderful magic black box which takes HTML, CSS and Javascript in one side and spits out native apps on the other side.

There are of course limitations to what you can do in a Cordova / PhoneGap Application and limitations in how and where you can create one. However, this cross-platform capability for building native apps with HTML, CSS and JavaScript is a tremendously powerful tool for web developers.

Cordova / PhoneGap has an additional JavaScript file which lets you write functions that speak to the native capabilities of smart phones and tablets.

# Creating a Cordova Project

# Installing Cordova | PhoneGap

Once you have installed your JDK and Android SDK (Installing Android Sdk) and Node.js then you are ready to install Cordova on your computer.

Android Platform Guide Official Reference

The Cordova install process has changed many times over the last several years. The current install is to use the Node Package Manager (NPM). As long as you have Node.js installed properly then you should be able install Cordova from the command prompt. Run the Command Prompt AS ADMINSTRATOR - open the Windows menu, type command, then right-click on the Command Prompt program shown and select Run as administrator.

C:\> npm install -g cordova
1

At the command prompt type the above. You should see NPM installing all the required bits to make Cordova run. After it is installed we will be using the command cordova to create projects, build them, and run them.

This is the node package manager downloading and installing the different files required by Cordova.

We have installed Cordova so that it can be accessed globally, from anywhere in our computer.

When complete you should be able to run the command

C:\> cordova -v
1

And see something like version 9.x.x.

Now we can see that we have Cordova installed and, more importantly which version we have.

# Creating a Project

With Cordova installed we can now create a new project.

Navigate to a folder where you want to create your project folder. Do NOT create the folder itself. Just navigate to the place where you want to create the folder.

On the command line type the following.

cordova create MyFirstApp ca.madd.abcd0001.first MyAppName
1

This will call the Cordova command to create your folder, the project structure and assign the name and id properties to your app in the project's new config.xml file.

After running the command you should see a success message and the new folder will exist.

There are three things we write after cordova create.

The first is the internal id for the project. This will be saved inside the config.xml file that stores your app's configuration. It will also become the new project folder.

cd firstCordovaApp
dir
1
2

These commands will take me inside the folder and show me the project structure.

The second string is the reverse-domain style identifier (namespace) for our new app. This style of naming is used to ensure uniqueness in our app names. When installing apps on devices this is the internal name. We need to make sure that this does not conflict with any other apps. Take a domain name from an imaginary website called madd.ca. Eg: http://abcd0001.madd.ca/. Then remove the http:// part and reverse the order -> ca.madd.abcd0001. Finally, add the name of your app to the end of this string.

Following this structure will ensure that your app never conflicts with or gets overwritten by anyone else's app in class.

The third string in the create options is the name for your app. This will be the visible name shown under the launcher icon on the device, after installation.

# Platforms and Plugins

When we want to add a specific platform, like Android or a plugin like the camera plugin then we use the cordova platform and cordova plugin commands.

# Plugins

The list of CORE Plugins

The list of additional plugins

Adding a plugin is very similar to adding a platform. We call the cordova command and pass in the options about which plugin to add to our project.

cordova plugin add cordova-plugin-battery-status
1

The last option in this command is the name of the plugin. The standard names for the plugins all start with cordova-plugin- followed by the descriptive name. All the core plugins follow this convention. Some plugins, written by independent developers or companies will have a differently formatted name.

If you want to search for plugins from the command line, you can do that too.

cordova plugin search keyword
1

To check which plugins are currently installed in your app:

cordova plugin ls
1
