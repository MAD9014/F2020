# Config.xml File

Here is a sample of the config.xml file:

<?xml version='1.0' encoding='utf-8'?>
<widget id="ca.edumedia.griffis.myapp" version="0.0.1" xmlns="http://www.w3.org/ns/widgets" xmlns:cdv="http://cordova.apache.org/ns/1.0">
  <name>TheApp</name>
  <description>
    A sample Apache Cordova application that responds to the deviceready event.
  </description>
  <author email="dev@cordova.apache.org" href="http://cordova.io">
    Apache Cordova Team
  </author>
  <content src="index.html" />
  <plugin name="cordova-plugin-whitelist" spec="1" />
  <access origin="*" />
  <allow-intent href="http://*/*" />
  <allow-intent href="https://*/*" />
  <allow-intent href="tel:*" />
  <allow-intent href="sms:*" />
  <allow-intent href="mailto:*" />
  <allow-intent href="geo:*" />
  <platform name="android">
    <allow-intent href="market:*" />
  </platform>
  <platform name="ios">
    <allow-intent href="itms:*" />
    <allow-intent href="itms-apps:*" />
  </platform>
</widget>

This is the configuration file for your Cordova project. It defines the internal name, the external name, and the version for your application.

You can edit this file directly to change the version of your app, the name that the user sees or the name that the phone will use to identify your app. The internal name is the id attribute.

The <name> element is the name the user sees.

The <author> element is something that you should edit to add your own name and email address.

The other elements in the config.xml file have to do with permissions for your application.

There are specific places in the file where certain setting elements need to be placed.
